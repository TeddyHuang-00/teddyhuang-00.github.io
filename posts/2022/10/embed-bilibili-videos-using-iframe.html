从 B 站直接获取的链接

在 B 站视频下方都能找到一个“分享”按钮，点击后选择“嵌入”，则可得到如下 iframe 代码：

<iframe
  src="//player.bilibili.com/player.html?aid=816173050&bvid=BV1nG4y1W73t&cid=850363308&page=1"
  scrolling="no"
  border="0"
  frameborder="no"
  framespacing="0"
  allowfullscreen="true"
>
</iframe>

一系列 iframe 属性的含义可以参考 MDN，这里我们不做赘述。

解读一下这段代码，其中 src 属性的值就是视频的链接，其中参数含义如下：

参数名 样值 说明
aid 816173050 视频 AV 号，可被 bvid 替代（已被 B 站弃用）
bvid BV1nG4y1W73t 视频 BV 号，可被 aid 替代
cid 850363308 chat id，可被 aid 或 bvid 替代
page 1 选集，下标从 1 记

这样的链接直接插入到博客中，效果非常不好，因为它的宽度和高度都是固定的，无法自适应，在大屏上显示效果很差；默认的清晰度也是最低的，而且在更改清晰度等操作时还会跳转到 B 站，因此我们需要稍作调整。

视频源问题（清晰度，自动播放等）

除了默认的这些 query 参数，B 站还提供了一些其他的参数，这里列举出一些用得到的参数：

参数名 样值 说明
as_wide 1 宽屏模式（0:小屏，1:宽屏）
high_quality 1 高质量模式（0:最低清晰度，1:最高清晰度）
可能已被禁用，实际显示效果似乎均仅限于 360P
danmaku 0 是否加载弹幕（0:不加载，1:加载）
t 0 默认开始时间（单位为秒，并且开启自动播放）

据此我们将代码修改为：

<iframe
  src="https://player.bilibili.com/player.html?bvid=BV1nG4y1W73t&page=1&as_wide=1&high_quality=1&danmaku=0&t=0"
  scrolling="no"
  border="0"
  frameborder="no"
  framespacing="0"
  allowfullscreen="true"
>
</iframe>

注意此处我们添加上了 https 前缀

适应页面大小

为了能够保持宽高比的同时占满整个列宽，我们需要使用一点 css 来完成这个任务。确保在你的博客所引用的 css 中包含了如下代码：

.aspect-ratio {
  position: relative;
  width: 100%;
  height: 0;
  padding-bottom: 75%;
}

.aspect-ratio iframe {
  position: absolute;
  width: 100%;
  height: 100%;
  left: 0;
  top: 0;
}

然后我们可以更改原先的 iframe 代码为

<div class="aspect-ratio">
  <iframe
    src="https://player.bilibili.com/player.html?bvid=BV1nG4y1W73t&page=1&as_wide=1&high_quality=1&danmaku=0&t=0"
    scrolling="no"
    border="0"
    frameborder="no"
    framespacing="0"
    allowfullscreen="true"
  >
  </iframe>
</div>

解决跳转问题

参考 MDN，iframe 提供了 sandbox 属性，我们可以用起放宽站点对于嵌入内容的限制，因此我们将代码修改为

<div class="aspect-ratio">
  <iframe
    src="https://player.bilibili.com/player.html?bvid=BV1nG4y1W73t&page=1&as_wide=1&high_quality=1&danmaku=0&t=0"
    scrolling="no"
    border="0"
    frameborder="no"
    framespacing="0"
    allowfullscreen="true"
    sandbox="allow-top-navigation allow-same-origin allow-forms allow-scripts"
  >
  </iframe>
</div>

其中 sandbox 内各选项的含义如下：

  • allow-top-navigation：允许嵌入资源导航到顶层上下文
  • allow-same-origin：允许嵌入资源以 same-origin 策略执行/加载
  • allow-forms：允许嵌入资源使用提交表格（可能会与一些交互选项操作有关）
  • allow-scripts：允许嵌入资源执行脚本（但不能创建弹出对话框）
