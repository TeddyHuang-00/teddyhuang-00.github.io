使用 GitHub Pages 和 Pelican 搭建个人博客

: 2022 / 10 / 06
: 2022 / 10 / 14
Nan Huang
: DevLog
: Python Web

尝试折腾了一下 GitHub Pages，发现自带的内容要么直接写 html 太简单粗暴，要么就是使用 Ruby 编写的 Jekyll 生成静态站点，对我来说不熟悉语言=可拓展性不强。因此记录一下使用 Pelican + GitHub Pages 搭建个人博客的过程，供以后参考。

Table of Contents
  • 安装 Pelican
  • 快速创建项目

  • 本地预览

Pelican 是一个静态网站生成器，使用 Python 编写，支持 Markdown 语法，可以使用 Markdown 编写文章，然后生成静态网页。而 GitHub Pages 只支持静态网页，所以使用 Pelican + GitHub Pages 搭建个人博客是一个不错的选择。

安装 Pelican

为了防止 Pelican 环境与主机环境冲突，我们可以新建一个虚拟环境来安装 Pelican

virtualenv pelican
source pelican/bin/activate

Pelican 支持使用 rst 或者 Markdown 语法编写文章，这里使用 Markdown 语法，因此我们需要安装 Markdown 相关的依赖。tzlocal 不是必要安装项，但是可以让 Pelican 正确检测时区，提供合适的默认选项。

pip3 install "pelican[markdown]" tzlocal

快速创建项目

在 GitHub 上创建一个新的仓库，需要以 <username>.github.io 的格式命名，如 teddyhuang-00.github.io ，然后 clone 到本地。这边使用的是 ssh 方法，需要预先配置好 ssh key 并上传公钥到 GitHub 服务器。

# replace with your user name
git clone git@github.com:TeddyHuang-00/teddyhuang-00.github.io.git
cd teddyhuang-00.github.io

在目录中我们使用 Pelican 提供的快速创建项目的命令来创建一个新的项目。

$ pelican-quickstart

这一步中会要求设置一些网站信息，比如站点名、作者名等等，可以根据自己的需要进行设置。需要注意的是最后配置发布时要确认选择 github pages 选项，会在 Makefile 文件中包含发布到 GitHub Pages 的命令。

此步之后项目内应该至少包含以下文件（夹）。

.
├── Makefile
├── content/
├── output/
├── pelicanconf.py
├── publishconf.py
└── tasks.py

编写文章

所有文章都放在 content 文件夹中，比如我们新建一个 Misc 文件夹，然后在其中新建一个 2022-10-06-My-first-post.md 文件，文件名格式为 YYYY-MM-DD-<title>.md ，其中 YYYY-MM-DD 为文章发布日期，<title> 为文章标题，中间用 - 连接。

mkdir content/Misc
touch content/Misc/2022-10-06-My-first-post.md

文章内容使用 Markdown 语法编写，可以参考 Markdown 语法说明。文章开头需要包含一些元数据，比如文章标题、发布日期、标签等等，这些元数据会在生成网页时被使用，一些常用的设置格式如下。

<!-- 大标题 -->
Title: 文章的标题
<!-- 创建时间 -->
Date: YYYY-MM-DD HH:SS
<!-- 修改时间 -->
Modified: YYYY-MM-DD HH:SS
<!-- 用于生成 url -->
Slug: 短链接别名
<!-- 用于归类文章 -->
Tags: 一些标签
<!-- 文稿状态，draft/hidden/published 其一 -->
Status: draft
<!-- 会显示在索引页上 -->
Summary: 文章摘要
<!-- 分类，缺省则根据所属子文件夹命名或为 misc -->
Category: Misc
<!-- 多个的话使用 Authors -->
Author: 作者

更多文稿元数据设置参见 Pelican 文档。在元数据之后，即可使用 Markdown 语法编写文章内容。

本地预览

完成内容编写之后，我们可以尝试在本地预览网站效果。在项目根目录下执行以下命令即可。

make devserver

这会调用 Pelican 来（重新）生成站点的静态文件到 output 文件夹中，并且开启监听模式，当文件发生变化时会自动重新生成。在浏览器中打开 http://localhost:8000 即可预览网站效果。

发布至 GitHub

在 Makefile 文件中包含了以下命令，可以用于发布网站到 GitHub Pages。

publish:
    "$(PELICAN)" "$(INPUTDIR)" -o "$(OUTPUTDIR)" -s "$(PUBLISHCONF)" $(PELICANOPTS)
github: publish
    ghp-import -m "Generate Pelican site" -b $(GITHUB_PAGES_BRANCH) "$(OUTPUTDIR)"
    git push origin $(GITHUB_PAGES_BRANCH)

其中使用 make github 时的逻辑为：

  1. 首先运行 Pelican 生成静态站点至 $(OUTPUTDIR) 文件夹
  2. 将 $(OUTPUTDIR) 文件夹内容发布到 $(GITHUB_PAGES_BRANCH) 分支
  3. 将 $(GITHUB_PAGES_BRANCH) 分支推送到远程仓库

这里我们希望使用默认的 main 分支用于开发，而将生成的静态站点发布到 gh-pages 分支。因此我们需要修改 Makefile 文件中的 GITHUB_PAGES_BRANCH 变量。

修改前

GITHUB_PAGES_BRANCH=main

修改后

GITHUB_PAGES_BRANCH=gh-pages

如果希望修改使用 Makefile 时的输出文件夹，同理修改 OUTPUTDIR 即可。

完成修改后，我们可以使用 make github 命令来发布网站。

make github

在 GitHub 仓库中，我们可以看到 gh-pages 分支已经被创建，并且包含了生成的静态站点文件。我们只需要手动将 gh-pages 分支设置为 GitHub Pages 的发布源分支即可。

在 Settings - Code and Automation - Pages - Build and deployment 中，我们将 Source 选择为 Deploy from a branch ，并且将 Branch 选择为 gh-pages 分支，文件夹保持 / (root) 即可，如下图所示。

完成更改之后点击 Save 即可。稍等片刻，GitHub Pages 就会自动部署我们的网站了。

更改主题模板

我们刚才新建的站点默认使用的是 notmyidea 主题，这是 Pelican 默认的主题。我们可以在 pelicanconf.py 文件中修改 THEME 变量来更改使用的主题。需要注意的是，由于 Pelican 安装时自带的仅有 notmyidea 和 basic 主题，因此我们需要手动安装其他主题并且指定路径。

在 Pelican themes 项目中，我们可以看到许多社区制作的主题。我们可以在这里找到自己喜欢的主题，然后将其克隆到本地。这里以 pelican-material 作为示例。

首先我们新建一个 themes 文件夹，用于存放主题，然后将想要使用的主题克隆到其中。

mkdir themes
cd themes
git submodule git@github.com:greizgh/pelican-material.git

然后我们修改 pelicanconf.py 文件，新增一个 THEME 变量，将其指向 themes/pelican-material 文件夹。

THEME = 'themes/pelican-material'

完成修改后，我们可以使用 make html 命令来生成网站，然后使用 make serve 命令来预览网站。或者如果你之前使用了 make devserver 来监听文件更改，那么你可以直接访问 http://localhost:8000 并刷新来预览网站更改。

如果你对于提供的主题并不满意，也可以尝试自己制作一个主题。在官方文档中给出了简单的说明供你开始。

添加 Giscus 支持

在搭建完网站主体框架之后，我们可以开始添加一些功能，比如这里即将添加的 Giscus 评论。

关于 Giscus 项目的介绍可以参考官方主页，简单来说，它是一个基于 GitHub Discussion API 构建的评论系统，能够在页面上嵌入一个评论框，用户可以使用 GitHub 账号进行评论。它的优点是不需要额外的服务器，而且可以直接使用 GitHub 的 Discussion 来管理评论，因此不需要额外的维护成本，并且也支持静态页面。

首先我们需要确保以下几点：

  1. 你的 GitHub 仓库是公开的，因为 Giscus 仅支持公开仓库。
  2. 你的 GitHub 仓库已经开启了 GitHub Discussion 功能，如果没有，在Settings - General - Features 下找到 Discussion 并勾选即可。
  3. 你的 GitHub 账号已经安装了 Giscus 应用。如果没有，可以直接点击这个链接安装 Giscus。

接下来我们只需要通过 Giscus 提供的配置工具来生成可以嵌入页面的代码，可配置的内容如下图

粘贴你的用户名/仓库名即可开始配置，我选用的选项如下：

  • 页面至 discussion 映射关系：URL（pathname 也可，这二者相对唯一，不易产生冲突）
  • 使用严格的标题匹配（使用哈希值来确保匹配结果准确）
  • Discussion 分类：Announcement（理论上新建一个类似于 Announcement 的上锁分类也可，这边我图方便直接使用 Announcement）
  • 只搜索该分类中的 discussion（缩小匹配范围，防止用户直接提交至仓库的内容也被显示）
  • 启用主帖子上的反应（好看）
  • 将评论框放在评论上方（方便用户输入评论）
  • 懒加载评论（节省用户网络资源，不必一次全部加载导致页面卡顿）

配置完成后你将会得到一条类似于下面的代码：

<script
  src="https://giscus.app/client.js"
  data-repo="[在此输入仓库]"
  data-repo-id="[在此输入仓库 ID]"
  data-category="[在此输入分类名]"
  data-category-id="[在此输入分类 ID]"
  data-mapping="pathname"
  data-strict="0"
  data-reactions-enabled="1"
  data-emit-metadata="0"
  data-input-position="bottom"
  data-theme="preferred_color_scheme"
  data-lang="zh-CN"
  crossorigin="anonymous"
  async
></script>

这里你可以选择将其手动置于 markdown 文件的末尾，或者如果你已经使用了非默认的主题的话，也可以像我一样修改主题 html 文件，将其置于合适的位置，比如在 /themes/<theme-name>/templates/base.html 中找到如下语句

{% block content %}
{% endblock %}

这个 jinja 语法对应的是页面的主体内容，将上面的代码粘贴到这之后即可。

为了防止本地预览的时候不小心提交一些测试评论，我们可以给 Giscus 添加额外的配置以限制特定 origins 内才加载 Giscus，这里一个简单的例子如下：

在根目录中新建一个名为 giscus.json 的文件，内容如下：

{
  "origins": ["https://teddyhuang-00.github.io"]
}

然后在 Makefile 中将 github 目标修改为：

github: publish
    cp giscus.json "$(OUTPUTDIR)/"
    ghp-import -m "Generate Pelican site $(shell date -u +"%Y-%m-%d %H:%M:%S")" -b $(GITHUB_PAGES_BRANCH) "$(OUTPUTDIR)"
    git push origin $(GITHUB_PAGES_BRANCH)

这样的话，在执行一次 make github 之后 giscus.json 会存在于发布分支的根目录中，因此只有在 teddyhuang-00.github.io 这个域名下的页面才会加载 Giscus，本地预览的时候就不会加载了。有关 Giscus 更进阶的配置，可以参考官方文档。

总结

在本文中，我们介绍了如何使用 Pelican 来生成一个静态站点，并且将其部署到 GitHub Pages 上。Pelican 作为一个静态站点生成器，其功能强大，使用简单。即便是萌新如我，折腾一晚上之后也能够顺利地搭建起一个属于自己的博客。如果你也想要搭建一个属于自己的博客，那么 Pelican 是一个不错的选择。
